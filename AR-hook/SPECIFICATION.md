# AR-Hook システム仕様書

## 1. システム概要

AR-Hook システムは、ドローン（ArduPilot搭載）に搭載された下向きカメラとモーションキャプチャシステム（Motive）を組み合わせ、ArUcoマーカー認識による精密な位置制御を実現するシステムです。

**構成要素:**
- **Pixhawk 6c** (フライトコントローラー、磁気センサー搭載)
- **Raspberry Pi 5** (ラズパイ、カメラ/通信処理)
- **下向きカメラ** (1640x1232解像度)
- **モーションキャプチャシステム (Motive)** (ドローン・荷物の姿勢取得)

---

## 2. 座標系と姿勢データ

### 2.1 座標系定義

| 座標系 | 説明 | 原点 | 軸方向 |
|------|------|------|------|
| **GPS座標系** | 緯度・経度・高度 | 基準点（REF_LAT, REF_LON, REF_ALT） | 度・メートル |
| **ローカル座標系** | 東・北・上（ENU） | GPS基準点をローカル原点に投影 | X(東), Y(北), Z(上) |
| **ドローン機体座標系** | 機体前・右・下 | ドローン重心 | X(前), Y(右), Z(下) |
| **カメラ座標系** | カメラ視点ベース | カメラ光学中心 | X(右), Y(下), Z(前=奥) |
| **荷物座標系** | 荷物中心ベース | 荷物中心 | X, Y, Z |

### 2.2 姿勢データソース

#### **ArduPilot から取得（MAVLink ATTITUDE メッセージ）**
```
Roll (φ):    -180° ～ +180°（機体X軸周り）
Pitch (θ):   -90° ～ +90°（機体Y軸周り）
Yaw (ψ):     0° ～ 360°（機体Z軸周り、磁気センサー+IMU統合）
```

**取得頻度:** 50Hz（MAVLink設定により調整可能）

**データ品質:**
- IMU（加速度・ジャイロ）で高精度
- 磁気センサーで長期ドリフト補正

#### **Motive から取得（UDP受信、クォータニオン形式）**
```
Quaternion: [qx, qy, qz, qw]
- qx, qy, qz: ベクトル成分
- qw: スカラー成分
- 正規化済み（|q| = 1）
```

**取得頻度:** 50Hz（UDP受信）

**データ品質:**
- 光学式モーションキャプチャの正確性
- カメラ視野内でのみ利用可能
- オフラインキャリブレーション可能

## 3. 姿勢フュージョン

### 3.1 フュージョン戦略

**入力:**
- Motive クォータニオン $q_{motive}$ （信頼度: 0.6～0.8）
- ArduPilot Roll, Pitch, Yaw（信頼度: 0.2～0.4）

**処理:**

1. **Motive クォータニオン → Euler角へ変換**
   $$
   q_{motive} \to (\phi_{motive}, \theta_{motive}, \psi_{motive})
   $$

2. **線形ブレンド**
   $$
   \phi_{fused} = w_{motive} \cdot \phi_{motive} + (1-w_{motive}) \cdot \phi_{AP}
   $$
   $$
   \theta_{fused} = w_{motive} \cdot \theta_{motive} + (1-w_{motive}) \cdot \theta_{AP}
   $$
   $$
   \psi_{fused} = w_{motive} \cdot \psi_{motive} + (1-w_{motive}) \cdot \psi_{mag}
   $$

   ※ $w_{motive}$ = 信頼度重み（デフォルト: 0.7）

3. **角度正規化**
   - Roll, Pitch: -180° ～ +180°
   - Yaw: 0° ～ 360°

**出力:**
- フュージョン済み姿勢角 $(\phi_{fused}, \theta_{fused}, \psi_{fused})$

### 3.2 フォールバック

| 状況 | 使用データ | 説明 |
|------|----------|------|
| Motive + ArduPilot 両方 | フュージョン値 | デフォルト（最高精度） |
| Motive のみ | Motive Euler | モーションキャプチャ環境外 |
| ArduPilot のみ | ArduPilot Roll/Pitch/Yaw | Motive 未接続時 |
| いずれも喪失 | 最後の有効値保持 | エラーハンドリング |

## 4. 座標変換パイプライン

### 4.1 カメラ座標系 → ワールド座標系への変換

**目的:** カメラが検出したマーカー位置（カメラ座標系）をドローン目標位置（ワールド座標系）に変換

**手順:**

#### **Step 1: カメラ座標系 → ドローン機体座標系**

カメラマウント定義（下向き固定カメラ）:
$$
x_{body} = -y_{cam}
$$
$$
y_{body} = x_{cam}
$$
$$
z_{body} = -z_{cam}
$$

#### **Step 2: ドローン機体座標系 → ワールド座標系（NED）**

回転行列構成（Euler角 → 回転行列）:
$$
R_{body \to earth} = R_z(\psi) \cdot R_y(\theta) \cdot R_x(\phi)
$$

ワールド座標系（NED：北・東・下）へ変換:
$$
\begin{bmatrix} X_{NED} \\ Y_{NED} \\ Z_{NED} \end{bmatrix} = R_{body \to earth} \cdot \begin{bmatrix} x_{body} \\ y_{body} \\ z_{body} \end{bmatrix}
$$

#### **Step 3: NED → NEU（北・東・上）への変換**

$$
X_{NEU} = Y_{NED} \quad \text{(東方向)}
$$
$$
Y_{NEU} = X_{NED} \quad \text{(北方向)}
$$
$$
Z_{NEU} = -Z_{NED} \quad \text{(上方向)}
$$

#### **Step 4: ドローン位置を原点とするオフセット**

$$
target_x = gps\_drone['x'] + X_{NEU}
$$
$$
target_y = gps\_drone['y'] + Y_{NEU}
$$
$$
target_z = gps\_drone['z'] + Z_{NEU}
$$

### 4.2 座標変換パイプル図

```
┌─────────────────────────────────────────────────────────┐
│  Motive + ArduPilot 姿勢フュージョン                      │
│  入力: quat_motive, roll_ap, pitch_ap, yaw_mag_ap      │
│  出力: φ_fused, θ_fused, ψ_fused                        │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  カメラ座標系 → 機体座標系                               │
│  tvec_cam = [x_cam, y_cam, z_cam]                      │
│  マウント変換:                                           │
│    x_body = -y_cam                                      │
│    y_body = x_cam                                       │
│    z_body = -z_cam                                      │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  ドローン機体座標系 → ワールド座標系（NED）              │
│  R = Rz(ψ) · Ry(θ) · Rx(φ)                             │
│  vec_NED = R · vec_body                                │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  NED → NEU（北・東・上）への変換                        │
│  X_NEU = Y_NED (東)                                     │
│  Y_NEU = X_NED (北)                                     │
│  Z_NEU = -Z_NED (上)                                    │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  ドローンGPS位置を原点とするオフセット                   │
│  target_x = gps_drone['x'] + X_NEU                     │
│  target_y = gps_drone['y'] + Y_NEU                     │
│  target_z = gps_drone['z'] + Z_NEU                     │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
        ╔═══════════════════╗
        ║  目標位置（ENU）  ║
        ║  target[x,y,z]    ║
        ╚═══════════════════╝
```

## 5. ArUco マーカー検出・追跡

### 5.1 マーカー配置

**目的:** 荷物（正方形）の3D位置・姿勢を推定

| マーカーID | 役割 | 配置 | 用途 |
|-----------|------|------|------|
| **1** | 中心マーカー | 正方形の中心 | ドローン位置の微調整基準 |
| **2** | 頂点マーカー | 右上（+X, +Y） | 荷物PnP推定用 |
| **3** | 頂点マーカー | 左上（-X, +Y） | 荷物PnP推定用 |
| **4** | 頂点マーカー | 左下（-X, -Y） | 荷物PnP推定用 |
| **5** | 頂点マーカー | 右下（+X, -Y） | 荷物PnP推定用 |

### 5.2 マーカー仕様

| 項目 | 値 |
|------|-----|
| 辞書 | DICT_4X4_50 |
| マーカーサイズ | 4 cm（一辺） |
| 正方形一辺長 | 15 cm |
| 半辺長（HALF_SIDE） | 7.5 cm |

### 5.3 検出アルゴリズム

**入力:** フレーム画像（1640x1232）

**処理:**
1. ArUco マーカー検出（`ArucoDetector`）
2. マーカーID判定
   - ID2-5: 荷物頂点マーカー → 3Dオブジェクト座標に変換
   - ID1: ドローン基準マーカー → 個別処理
3. PnP推定（`cv2.solvePnP`）
   - 入力: 3Dオブジェクト点群、2D画像点群
   - 出力: 回転ベクトル（rvec）、平行移動ベクトル（tvec）
   - フラグ: `cv2.SOLVEPNP_ITERATIVE`

**出力:**
- `center_cam`: 荷物中心のカメラ座標系位置 [x, y, z]
- `rvec_cargo`: 荷物の回転ベクトル（Rodrigues形式）

### 5.4 フォールバック処理

```
┌─ 全マーカー（ID2-5）検出 ─┐
│  └─ PnP推定成功             │
│     └─ center_cam 確定       │
│        最後の有効値として保持 │
└────────────────────────────┘

        or

┌─ 頂点マーカー見失い ─┐
│ ID1 のみ検出           │
│ └─ 最後の center_cam   │
│    を使用（継続追跡）   │
└────────────────────────┘

        or

┌─ 全マーカー喪失 ─┐
│ キーボード制御へ  │
│ フォールバック     │
└──────────────────┘
```

## 6. ID1-中心間距離計算

### 6.1 目的

ドローン位置微調整の基準となる、ID1（ドローン側マーカー）と荷物中心（ID2-5推定値）間の相対距離を計算

### 6.2 計算式

**Step 1: ベクトル計算**
$$
\vec{v}_{cam} = \vec{p}_{ID1} - \vec{p}_{center}
$$

**Step 2: 荷物座標系への変換**
$$
\vec{v}_{cargo} = R_{cargo}^T \cdot \vec{v}_{cam}
$$

ここで $R_{cargo}$ は PnP で推定された回転行列（Rodrigues形式から変換）

**Step 3: 距離出力**
$$
dist\_x = v_{cargo}[0], \quad dist\_y = v_{cargo}[1], \quad dist\_z = v_{cargo}[2]
$$

### 6.3 用途

- **誘導精度向上:** ドローンの微妙な位置ずれ補正
- **診断:** ID1 が常に想定位置にあるか確認
- **CSV記録:** 運用ログ分析用

### 7.1 目標値決定フロー


### 7.2 誘導パラメータ

| パラメータ | 値 | 説明 |
|----------|-----|------|
| **高度目標** | 1.10 m | `TARGET_HEIGHT_ABOVE_TAKEOFF` |
| **ヨー角制御** | 手動 or 自動 | キーボード or 目標ヨー角 |
| **離陸高度** | 0.50 m | `TAKEOFF_ALT` |
| **送信周期** | 100 ms | 10 Hz（`SEND_HZ=10`） |

### 7.3 誘導コマンド（MAVLink）

**メッセージ名:** `SET_POSITION_TARGET_GLOBAL_INT`

```
├─ Frame: GLOBAL_RELATIVE_ALT_INT
├─ Lat/Lon: GPS座標 (int, degE7)
├─ Alt: 高度（相対） (float, m)
├─ Yaw: ヨー角制御 (float, rad)
└─ 速度/加速度: 0（位置制御のみ）
```

**付加メッセージ:** `CONDITION_YAW`
- ヨー角スムーズ制御
- 角速度: 20°/s

## 9. 通信仕様

### 9.1 ArduPilot-ラズパイ間（MAVLink）

| 項目 | シリアル接続 | USB接続 |
|------|-----------|--------|
| **ポート** | `/dev/ttyAMA0` | `/dev/ttyACM0` |
| **ボーレート** | 1000000 bps | 115200 bps |
| **制御線** | RTS/CTS 有効 | なし |
| **メッセージタイプ** | HEARTBEAT, ATTITUDE, GLOBAL_POSITION_INT, GPS_INPUT（受信）, SET_POSITION_TARGET_GLOBAL_INT, CONDITION_YAW（送信） |
| **通信周期** | 50 Hz（HEARTBEAT など定期）, 10 Hz（誘導コマンド） |

### 9.2 Motive-ラズパイ間（UDP）

| 項目 | 値 |
|------|-----|
| **プロトコル** | UDP |
| **ポート** | 15769 |
| **送信元** | Motive PC |
| **受信側** | ラズパイ（0.0.0.0:15769） |
| **ペイロード** | バイナリ固定長（39バイト） |
| **形式** | `<BiiiHfffffd` |

**バイナリペイロード構成:**

```c
struct MotiveGPSData {
    uint8_t rigid_body_id;    // 1 byte
    int32_t lat_e7;           // 4 bytes (degE7)
    int32_t lon_e7;           // 4 bytes (degE7)
    int32_t alt_mm;           // 4 bytes (mm)
    uint16_t yaw_cdeg;        // 2 bytes (centidegrees)
    float32_t qx, qy, qz, qw; // 16 bytes (quaternion)
    float64_t unix_time_sec;  // 8 bytes
};
```

**受信周期:** 50 Hz（Motive フレームレート依存）

## 12. エラーハンドリング

### 12.1 重大エラー

| エラー | 対応 |
|------|------|

## 15. 制限事項と注意事項

### 15.1 システム制限

| 項目 | 制限値 | 理由 |
|------|-------|------|
| **マーカー検出距離** | ~3 m（最大） | カメラ解像度 |
| **フレームレート** | 20 FPS | Picamera2 |
| **誘導周期** | 10 Hz | MAVLink 帯域幅 |
| **姿勢フュージョン精度** | ±2° | Motive キャリブレーション |
| **ドローン傾斜角** | ±30°（推奨） | カメラ視野確保 |

### 15.2 環境要件

- **屋外:** GPS信号必須（MAVLink GPS統合）
- **屋内:** Motive モーションキャプチャ必須
- **照度:** カメラ：500 lux 以上（ArUco検出精度向上）
- **無線干渉:** 2.4 GHz 帯域避け推奨

### 15.3 運用注意事項

1. **起動時チェック:**
   - MAVLink ハートビート確認
   - 磁気センサーキャリブレーション済み
   - Motive UDP 受信確認（UDP接続時）

2. **飛行中:**
   - ArUco マーカー常時視認性確保
   - 電池残量監視
   - ドローン傾斜≤30° 維持

3. **安全:**
   - 周囲安全距離確保（3 m 以上）
   - Emergency Stop キー 'q' 即座実行可能に

## 16. トラブルシューティング

### 16.1 よくある問題

| 問題 | 原因 | 解決方法 |
|------|------|--------|
| マーカー検出されない | 照度不足 or マーカー外観損傷 | 照度UP / マーカー交換 |
| Yaw角ドリフト | 磁気センサー未キャリブレーション | Pixhawk キャリブレーション実行 |
| 位置精度低い | Motive クォータニオン信頼度低 | フュージョン重み調整 (0.5～0.9) |
| フレームレート低下 | CPU 負荷過高 | ビデオ出力無効化 |
| UDP 受信失敗 | ネットワーク遅延 or ファイアウォール | ポート 15769 確認 |

### 16.2 デバッグコマンド

```bash
# MAVLink 通信テスト
mavproxy.py --master=/dev/ttyAMA0 --baud=1000000

# UDP 受信テスト
nc -ul 15769

# CPU/メモリ監視
top -b -n 1
```
